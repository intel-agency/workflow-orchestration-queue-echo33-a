# AI Repository Summary

This document provides a comprehensive summary of the workflow-orchestration-queue repository for AI agents and automated systems.

## Repository Identity

- **Name:** workflow-orchestration-queue-echo33-a
- **Type:** GitHub Template Repository
- **Purpose:** Headless agentic orchestration platform
- **Primary Language:** Python 3.12+
- **Framework:** FastAPI

## Core Purpose

Transform GitHub Issues into automated execution orders for AI agents. The system replaces interactive AI coding patterns with persistent, event-driven infrastructure that autonomously fulfills specification-driven tasks without human intervention.

## Architecture Summary

### Four Pillars

1. **The Ear (Notifier)** - `src/orchestration_queue/notifier_service.py`
   - FastAPI webhook receiver
   - HMAC SHA256 verification
   - Event triaging

2. **The State (Queue)** - `src/orchestration_queue/queue/github_queue.py`
   - GitHub Issues as state persistence
   - ITaskQueue abstraction layer
   - Distributed locking via assignees

3. **The Brain (Sentinel)** - `src/orchestration_queue/orchestrator_sentinel.py`
   - Background polling service
   - Task claiming (assign-then-verify)
   - Shell-bridge execution

4. **The Hands (Worker)** - External opencode CLI
   - Isolated DevContainer execution
   - AI workflow automation

### Key Models

- **WorkItem** (`src/orchestration_queue/models/work_item.py`)
  - Unified task representation
  - TaskType enum: PLAN, IMPLEMENT, BUGFIX
  - WorkItemStatus enum mapping to GitHub labels

- **GitHub Events** (`src/orchestration_queue/models/github_events.py`)
  - Webhook payload schemas
  - Pydantic validation

## Environment Variables

### Required
- `GITHUB_TOKEN` - GitHub PAT for API access
- `GITHUB_ORG` - Organization/user name
- `GITHUB_REPO` - Repository name

### Recommended
- `WEBHOOK_SECRET` - HMAC verification
- `SENTINEL_BOT_LOGIN` - Task claiming username

### Optional
- `LOG_LEVEL` - Logging verbosity
- `POLL_INTERVAL` - Sentinel polling frequency
- `HEARTBEAT_INTERVAL` - Progress update frequency

## Development Workflow

1. **Install dependencies:** `uv pip install -e ".[dev]"`
2. **Run linter:** `ruff check src/`
3. **Run type checker:** `mypy src/`
4. **Run tests:** `pytest`
5. **Build container:** `docker build -t orchestration-queue .`

## Integration Points

### GitHub Webhooks
- Endpoint: `POST /webhooks/github`
- Events: `issues` (opened, labeled)
- Labels: `orchestration:*`, `implementation:ready`

### Shell Bridge
- Script: `scripts/devcontainer-opencode.sh`
- Commands: `up`, `start`, `prompt`, `stop`

## Security Considerations

- Webhook signatures verified via HMAC SHA256
- Credentials scrubbed from public logs
- Tokens injected as ephemeral environment variables
- Worker containers isolated in Docker network
